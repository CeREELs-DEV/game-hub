# 카드 업데이트 날짜 표기 디자인

## 목표

게임 허브의 각 카드 타이틀 바로 아래에 해당 프로젝트의 최근 업데이트 날짜를 표시한다. 날짜는 기존 카드의 `LIVE`, `PLAY` 메타 정보와 같은 시각 언어를 사용하면서도 타이틀보다 낮은 정보 위계를 유지해야 한다.

## 검토한 방식

### A. 라벨 + 숫자 날짜

`UPDATED · 2026.07.13`

- 장점: 짧고 빠르게 읽히며 카드별 폭이 일정하다.
- 장점: 기존 영문 대문자 및 모노스페이스 메타 정보와 일관된다.
- 단점: 한국어 사용자가 처음 볼 때 `UPDATED`의 의미를 해석해야 한다.

### B. 영문 에디토리얼형

`LAST UPDATED — 13 JUL 2026`

- 장점: 영문 편집 디자인의 인상이 강하다.
- 단점: 텍스트가 길어 좁은 카드와 긴 타이틀 아래에서 시각적 밀도가 높다.

### C. ISO 날짜 우선형

`2026-07-13 · UPDATE`

- 장점: 정렬과 데이터 해석이 명확하다.
- 단점: 사용자용 카드보다 시스템 로그처럼 보일 수 있다.

## 결정

A안인 `UPDATED · YYYY.MM.DD`를 사용한다. 허브에 이미 사용 중인 `IBM Plex Mono`, 대문자 영문 메타 텍스트, `LIVE`/`PLAY` 표기와 가장 자연스럽게 연결되기 때문이다.

상대 날짜(`3 DAYS AGO`)는 시간이 지나면 표시 의미가 달라지고 정적 페이지에서 자동 갱신되지 않으므로 사용하지 않는다.

## 표시 위치와 스타일

- 각 `<h2 class="title">` 바로 다음에 날짜를 배치한다.
- 시맨틱 HTML인 `<time>` 요소를 사용한다.
- 화면 텍스트는 `UPDATED · YYYY.MM.DD`, `datetime` 속성은 표준 `YYYY-MM-DD`를 사용한다.
- 글꼴은 `IBM Plex Mono`, 크기는 `10px`, 굵기는 `500`, 자간은 `.08em`으로 한다.
- 색상은 보조 텍스트 토큰인 `--ink-soft`를 사용한다.
- 타이틀과 날짜 사이는 `8px`, 날짜와 하단 CTA 영역은 기존 유연 공간을 유지한다.
- 타이틀의 기존 `margin-bottom`은 제거하고 날짜 요소가 하단 여백을 담당하도록 한다.

예시:

```html
<h2 class="title">STORY GOGGLES</h2>
<time class="updated" datetime="2026-07-13">UPDATED · 2026.07.13</time>
```

## 날짜 산정 규칙

날짜는 각 카드가 가리키는 프로젝트 기본 브랜치의 최신 커밋 시각을 한국 표준시(`Asia/Seoul`)의 날짜로 변환한 값이다. 허브 내부 프로젝트는 현재 저장소의 해당 디렉터리를 마지막으로 변경한 커밋을 사용한다.

| 카드 | 기준 저장소/경로 | 표시 날짜 |
| --- | --- | --- |
| STORY GOGGLES | `cereels-dev/two` 기본 브랜치 최신 커밋 | `2026.07.13` |
| MATTER OF PERSPECTIVE | `cereels-dev/literary` 기본 브랜치 최신 커밋 | `2026.07.07` |
| Life is a Fairy Tale | 현재 저장소 `demo/` 마지막 변경 커밋 | `2026.07.09` |
| Connections | 현재 저장소 `connections/` 마지막 변경 커밋 | `2026.06.12` |
| Coupang | `cereels-gamedev/coupang` 기본 브랜치 최신 커밋 | `2026.06.12` |
| Escape Room | `cereels-gamedev/escape-room` 기본 브랜치 최신 커밋 | `2026.06.12` |

날짜는 빌드 시 자동 조회하지 않고 HTML에 명시한다. 현재 허브는 빌드 파이프라인 없는 정적 문서이므로 런타임 API 호출이나 토큰 관리 복잡도를 추가하지 않는다. 프로젝트가 배포될 때 담당자가 해당 카드 날짜도 함께 갱신한다.

## 접근성 및 동작

- 날짜는 카드 링크 내부의 일반 텍스트로 노출되어 스크린 리더에서도 타이틀 다음에 읽힌다.
- `datetime` 속성에는 기계 판독 가능한 ISO 날짜를 제공한다.
- 날짜 자체는 별도 링크나 버튼으로 만들지 않는다.
- 모바일에서는 카드가 세로로 쌓이므로 별도 축약이나 줄바꿈 규칙이 필요하지 않다.

## 검증

- 모든 6개 카드에 타이틀 바로 다음 날짜가 존재하는지 테스트한다.
- 각 `<time>`의 화면 텍스트와 `datetime` 값이 서로 일치하는지 확인한다.
- 기존 카드 순서, 링크, `LIVE` 상태, CTA가 변하지 않았는지 회귀 테스트한다.
- 데스크톱과 모바일 폭에서 날짜가 한 줄로 표시되고 CTA를 침범하지 않는지 확인한다.

## 후속 개선 검토

1. 날짜를 `<time>`으로 표현해 접근성과 기계 판독성을 높인다.
2. 날짜 산정 기준과 시간대를 명시해 담당자마다 다른 날짜를 입력하는 문제를 막는다.
3. 카드별 날짜 존재 여부와 형식을 자동 테스트해 누락과 오타를 방지한다.

세 항목 모두 구현 범위가 작고 데이터 신뢰성을 직접 높이므로 이번 변경에 함께 반영한다.
