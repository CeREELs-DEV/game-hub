# Story Goggles 카드 순서 변경 설계

## 목표

게임 허브에서 `STORY GOGGLES` 카드를 첫 번째에 배치한다.

## 변경 범위

- `STORY GOGGLES` 카드의 HTML DOM 위치를 첫 번째로 이동한다.
- `STORY GOGGLES`의 표시 번호를 `01`로 변경한다.
- `MATTER OF PERSPECTIVE`를 두 번째로 이동하고 표시 번호를 `02`로 변경한다.
- 나머지 네 카드의 순서, 번호, 링크 및 콘텐츠는 유지한다.
- 카드 순서를 검증하는 테스트 기대값을 새 순서에 맞춘다.

## 제외 범위

- `/two/` 저장소 또는 배포물은 수정하지 않는다.
- `STORY GOGGLES` 링크인 `https://cereels-dev.github.io/two/`는 변경하지 않는다.
- 사용자가 지정했던 `20260713_Story Goggles_v5.html`은 이 저장소에 복사하지 않는다.
- 카드 이미지와 스타일은 변경하지 않는다.

## 적용 방법 검토

1. HTML DOM 순서 변경: 시각적 순서, 키보드 탐색 순서, 스크린리더 읽기 순서가 일치한다. 이 방법을 사용한다.
2. CSS `order` 사용: 화면 순서만 바뀌고 접근성 탐색 순서가 기존 DOM과 달라질 수 있어 사용하지 않는다.
3. JavaScript 런타임 재배치: 정적 허브에 불필요한 실행 의존성과 복잡성을 추가하므로 사용하지 않는다.

## 검증

- Node 테스트에서 카드 제목, 링크 및 이미지 순서가 새 요구사항과 일치하는지 확인한다.
- 정적 HTML에서 카드 번호가 `01`부터 `06`까지 중복 없이 이어지는지 확인한다.
- 브라우저에서 첫 번째 카드가 `STORY GOGGLES`, 두 번째 카드가 `MATTER OF PERSPECTIVE`로 표시되는지 확인한다.
